# 🏦 경희대학교 - 데이터 분석 캡스톤 디자인 (Data Analysis Capstone Design)

이 저장소는 **경희대학교 데이터 분석 캡스톤 디자인** 프로젝트를 위한 공간입니다.  
본 프로젝트는 기존 금융권에서 소외된 **신파일러(Thin Filer)**를 대상으로, **행동 데이터 기반의 설명 가능한 신용 평가 및 맞춤형 페르소나 구축**을 핵심 목표로 합니다.

---

## 🎯 프로젝트 개요
*   **주제**: 비금융(행동) 데이터를 활용한 신파일러 신뢰도 추론 및 LLM 기반 설명 가능한(XAI) 금융 서비스 구축
*   **비전**: 공급자 중심의 '상품 판매'에서 사용자 중심의 '맥락 있는 서비스'로 금융 패러다임 전환
*   **핵심 가치**: 포용적 금융(Inclusive Finance), 리스크 관리, 사용자 신뢰 확보

## 🔍 문제 정의 및 배경 (3주차 업데이트)
*   **금융권의 과제**: 전통적 신용평가 시스템은 금융 이력이 부족한 **신파일러(대학생, 소상공인, 저신용자 등)**를 잠재적 위험군으로 분류하여 배제하고 있음.
*   **전환 전략**: 
    *   기존 우량 고객 유치 경쟁보다는, 새로운 고객층인 신파일러를 확보하여 장기적인 신뢰 관계를 구축.
    *   신파일러의 낮은 충성도와 이탈률 문제를 해결하기 위해, 단순 점수 부여가 아닌 **'설명 가능한 신뢰'**를 제공하여 사용자 경험을 강화함.
*   **기존 접근의 한계**: 점수 중심의 평가 방식은 사용자에게 결과의 원인을 설명하지 못하며, 이는 금융 서비스에 대한 불신으로 이어짐.

## 💡 해결 방안 및 차별점
1.  **대안 데이터(Alternative Data) 활용**: 금융 기록 대신 행동 데이터 및 비금융 데이터를 기반으로 사용자의 '상환 신뢰도'를 추론.
2.  **LLM 기반 페르소나 구축**: 사용자의 소비 패턴과 행동 데이터를 LLM으로 분석하여 고유의 '금융 페르소나'를 생성.
3.  **설명 가능한 신뢰(Explainable Trust)**: 신뢰도 점수의 근거를 LLM을 통해 내러티브(Narrative)하게 설명함으로써 사용자 중심의 인터페이스 제공.

## 🛠️ 분석 및 구현 프로세스
1.  **과제 정의**: 금융권 리스크 관리 및 신뢰 기반의 사용자 중심 시스템 정의
2.  **데이터 설계/가공**: 
    *   캐글(Kaggle) 및 비금융 행동 데이터를 활용한 신파일러 데이터셋 재구성
    *   한국 금융 환경을 반영한 가상 페르소나 시나리오 작성
3.  **모델링 및 분석**:
    *   군집 분석(K-means)을 통한 사용자 그룹화 및 페르소나 정의
    *   LLM API 연동을 통한 신뢰도 근거 생성 모델 구축
4.  **시스템 구현 및 평가**: 페르소나 기반 시뮬레이션을 통한 모델의 적절성 및 설명력 검증
